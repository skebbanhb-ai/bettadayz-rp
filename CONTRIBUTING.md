Contribution guidelines

Thanks for your interest in contributing to BettaDayz RP!

- To update the published site, edit files under `public/` and push to `main`.
- For site-level changes, run a local preview (open `public/index.html` in a browser).
- For code/contributor changes:
  - Fork the repo, create a feature branch, and open a pull request against `main`.
  - Include a short description and testing instructions in your PR description.
- Tests: run `npm test` for the test suite in `tests/`.
- Keep changes small and focused. If your change affects the deployed site, mention it in the PR title.

Maintainers: Please review PRs and merge when CI passes and review is complete.
